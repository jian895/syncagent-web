快速开始

1. 登录账号

访问 安装页面，用 Google 账号一键登录，30 秒完成。

2. 安装 MCP

复制安装命令，粘贴给你的 AI 助手：

帮我安装这个 MCP 服务器：
{
  "syncagent": {
    "url": "https://web-production-1fa7e.up.railway.app/mcp",
    "headers": {
      "Authorization": "Bearer your_token_here"
    }
  }
}

3. 重启智能体

重启后即可对 AI 说「备份我的配置」或「恢复我的配置」。

功能说明

养成资产备份

对 AI 说：「备份我的配置」

系统按该智能体的清单收集并上传：

  • 技能（SKILL.md 等）
  • 记忆（memory / MEMORY.md 等）
  • 人格设定（SOUL / IDENTITY / USER 等）
  • 规则与项目指令（rules / AGENTS.md）

不备份：API Key、Token、MCP 连接配置、登录态、运行缓存。密钥永远本机配置。

养成资产恢复

在新设备上安装 MCP 后，对 AI 说：「恢复我的配置」

系统会：

  • 返回结构化写回计划（每个文件的路径 + 内容）
  • 由 AI 按 relpath 精确写回本地目录
  • MCP / 密钥类配置需你在本机自行配置

当前不做的事

  • 跨智能体自动迁移（如 Cursor → Hermes 格式转换）— 尚未实现
  • 密钥云同步 / MCP 网关 — 规划中，暂缓

支持的智能体

智能体 同步内容 状态
Reasonix skills / memory / AGENTS.md ✅ 已校准
Hermes skills / memory / SOUL / AGENTS.md ✅ 已校准
Codex skills / rules / AGENTS.md（memory 为 SQLite，暂不支持） ✅ 已校准
WorkBuddy skills / memory / SOUL / IDENTITY / USER ✅ 已校准
Cursor / Windsurf 项目规则（.cursorrules / .windsurfrules 等） ⚠️ 未完全校准

Claude Desktop 为云原生客户端，养成资产在服务端，本地无可同步内容，暂不支持。

常见问题

Q: API Key 会上传到云端吗？

A: 不会。MCP / 模型 / 连接器等含密钥的配置根本不在备份清单里。只同步技能、记忆、人格这类纯文本养成资产。恢复后密钥在本机自行配置。

Q: 为什么不备份 MCP 配置？

A: MCP 配置的价值几乎全在 key 上；结构本身是公开模板。密钥不能可靠经 AI 上传（会被安全对齐打码），因此统一本机配置。未来可能通过「MCP 网关」在平台托管 key，那是另一条路线。

Q: 支持跨智能体迁移吗？

A: 当前只做同智能体换设备（如 Reasonix → Reasonix）。跨产品格式转换尚未实现。

Q: 配置文件存在哪里？

A: Cloudflare R2（S3 兼容），按用户隔离；仅持有你 JWT 的请求可读。

Q: Codex 的记忆为什么同步不了？

A: Codex 记忆是 SQLite 二进制库，当前架构只同步纯文本。skills / rules / AGENTS.md 可同步；memory 标为暂不支持。

Q: 遇到问题怎么办？

A: 访问 GitHub Issues 提交问题。
